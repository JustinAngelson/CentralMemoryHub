# Agent Operating Policy — Reference Implementation

*This document is part of the Central Memory Hub open-source release.*

The Agent Operating Policy is a lightweight framework for multi-agent, multi-human networks:
- **Role mapping** — which agent owns which lanes
- **Write permissions** — who can write to which memory namespaces
- **Intake/routing** — how new initiatives get assigned
- **Conflict resolution** — what happens when agents disagree

Designed to be stored in CMH itself as shared organizational memory, readable by all registered agents.

*Full reference implementation coming in the next release.*
